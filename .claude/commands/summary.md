# Summary Protocol

## 1. Find changes
- Figure out all the changes made since the last git commit. 

## 2. Summarize 
- Use a ~6 word heading that gives a sneak peak into the changes 
- Follow with a body (max 800 chars) with each change as a separate paragraph.
- No need to describe exact line numbers or file names.
 
## 3. Confirm
Show me the summary so I can verify.
