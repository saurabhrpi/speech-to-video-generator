19. **No `CustomerInfo` listener — offline-replay + RC-ingestion-lag gap** (S49 Case 1a, originally surfaced S48). Our app calls `/api/credits/grant` only from `Paywall.handlePurchase` (when `purchasePackage` resolves) and `Paywall.handleRestore` (manual tap). Real App Store `StoreKit.PaymentQueue` replays transactions that failed mid-flight (offline, crash, etc.) via its observer API, and RC emits these through `CustomerInfo` updates. Without a listener, replayed transactions are silently lost until the user manually taps Restore. Fix: add `Purchases.addCustomerInfoUpdateListener(info => ...)` in `configurePurchases()` (`mobile/lib/purchases.ts`) that diffs `nonSubscriptionTransactions` against a locally-tracked set of already-granted txIds and calls `grantCreditsForTransaction` (idempotent server-side) for each new matching tx. Also closes the S48 "RC Test Store REST delay" gap (`memory/reference_rc_test_store_rest_delay.md`) — auto-grants after ingestion catches up instead of requiring Restore tap.
