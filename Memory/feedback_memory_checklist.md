Before every code change, scan the Feedback section one-liners in MEMORY.md as an active gate. Don't skip this when heads-down in code.

**Why:** Re-reading all feedback files before every change balloons the context window. But ignoring the rules entirely leads to misses. MEMORY.md is already loaded — the one-liners ARE the checklist at zero extra cost.

**How to apply:** Before writing any code, mentally check each Feedback one-liner in MEMORY.md. Only Read the full feedback file if the one-liner isn't enough to recall the details. This replaces the earlier plan of re-reading all files before every change.
